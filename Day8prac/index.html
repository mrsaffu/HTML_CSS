1. What is React JS
  • ReactJS is a JavaScript library that is used to build user interfaces (UIs) for web, mobile, and desktop applications. It is one of the most popular tools for front-end web development.
  • ReactJS uses a virtual DOM (JavaScript object) to render components, which are small, isolated pieces of code. This makes apps more efficient and faster than the standard DOM.
  • ReactJS is a good choice for building single-page or mobile apps.
  1. How it works

  • React.js uses components, which are small, reusable pieces of code that output HTML.
    When data changes, React.js updates and re-renders only the components that need to change, which makes the user experience faster.

  1. What is MongoDB?
    MongoDB is a popular, open-source, NoSQL database management system designed for handling large volumes of unstructured or semi-structured data. It uses a document-oriented data model, which means data is stored in flexible, JSON-like documents.
  1. Key Features:
  • Schema-less: Unlike traditional relational databases, MongoDB allows you to store data in a flexible schema. This makes it easy to adapt to changing requirements.
  • Document Storage: Data is stored in BSON (Binary JSON) format, enabling rich data structures. Each document can have its own unique structure.
  • Scalability: MongoDB is designed to scale out by distributing data across multiple servers, which allows for horizontal scaling.
  • Performance: It offers high read and write performance, making it suitable for high-load applications.
  • indexing MongoDB supports various indexing methods to optimize query performance, including single-field, compound, and geospatial indexes.
  1. Where do we use MongoDB?
    2. MongoDB is preferred over RDBMS in the following scenarios:
  • Big Data: If we have huge amount of data to be stored in tables, think of MongoDB before RDBMS databases. MongoDB has built-in solution for partitioning and sharding our database.
  • Unstable Schema: Adding a new column in RDBMS is hard whereas MongoDB is schema-less. Adding a new field does not effect old documents and will be very easy.
  • Distributed data : Since multiple copies of data are stored across different servers, recovery of data is instant and safe even if there is a hardware failure.
  1. Language Support by MongoDB
  • MongoDB currently provides official driver support for all popular programming languages like C, C++, Rust, C#, JAVA, Node.js, Perl,PHP, Python, Ruby, Scala, Go and Erlang.
